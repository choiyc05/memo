### 1. AES-256 (개인정보 암호화: cryptography)
`cryptography` 라이브러리를 설치하고, `Fernet`을 이용해 대칭키 암호화를 구현.

**구현 단계:**
* **설치:** `uv add cryptography`
* **용도:** DB에 저장되는 전화번호, 주소, 실명 등 복호화가 필요한 데이터.

```python
from cryptography.fernet import Fernet
import os

# 1. 키 생성 (최초 1회만 실행해서 .env에 저장하세요)
# key = Fernet.generate_key() 

# 2. .env에서 키 로드
SECRET_KEY = os.getenv("DB_ENCRYPTION_KEY")
fernet = Fernet(SECRET_KEY)

def encrypt_data(plain_text: str) -> str:
    return fernet.encrypt(plain_text.encode()).decode()

def decrypt_data(cipher_text: str) -> str:
    return fernet.decrypt(cipher_text.encode()).decode()

# FastAPI 엔드포인트 예시
@app.post("/user/profile")
async def create_profile(phone: str, db: Session = Depends(get_db)):
    encrypted_phone = encrypt_data(phone)
    # DB에는 encrypted_phone 저장
```

### 2. Password Hashing (비밀번호 해싱: `passlib`, `bcrypt`)
비밀번호는 복호화가 불가능한 '단방향 해시' 사용.

**구현 단계:**
* **설치:** `uv add "passlib[bcrypt]"`
* **용도:** 사용자의 비밀번호 저장 및 로그인 검증.

```python
from passlib.context import CryptContext

# bcrypt 알고리즘 사용 설정
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain_password, hashed_password) -> bool:
    return pwd_context.verify(plain_password, hashed_password)

# 로그인 시 사용 예시
if not verify_password(login_data.password, user_in_db.hashed_password):
    raise HTTPException(status_code=400, detail="비밀번호가 틀렸습니다.")
```

### 3. RSA (비대칭키 암호화: 비공개 데이터 교환)
RSA는 공개키(Public Key)로 암호화하고 개인키(Private Key)로 복호화합니다. 팀 프로젝트에서 특정 관리자만 열어봐야 하는 파일을 전송하거나, 클라이언트-서버 간에 키를 교환할 때 사용합니다.

* **설치:** `uv add pycryptodome`
* **용도:** 디지털 서명, 아주 중요한 소량의 데이터 암호화(비밀키 교환 등).

```python
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_OAEP
import binascii

# 1. 키 쌍 생성 (실제로는 생성 후 파일/환경변수로 관리)
key = RSA.generate(2048)
private_key = key.export_key() # 서버 비밀 금고 보관
public_key = key.publickey().export_key() # 외부에 공개 가능

# 2. 공개키로 암호화 (누구나 할 수 있음)
def rsa_encrypt(plain_text: str, pub_key_str: bytes):
    recipient_key = RSA.import_key(pub_key_str)
    cipher_rsa = PKCS1_OAEP.new(recipient_key)
    return binascii.hexlify(cipher_rsa.encrypt(plain_text.encode())).decode()

# 3. 개인키로 복호화 (서버 주인만 가능)
def rsa_decrypt(cipher_text: str, priv_key_str: bytes):
    private_key = RSA.import_key(priv_key_str)
    cipher_rsa = PKCS1_OAEP.new(private_key)
    return cipher_rsa.decrypt(binascii.unhexlify(cipher_text)).decode()

# 사용 예시
secret_msg = "Admin-Only-Message"
encrypted = rsa_encrypt(secret_msg, public_key)
decrypted = rsa_decrypt(encrypted, private_key)
```

### 4. HMAC-SHA256 (JWT 서명)

* **설치:** `uv add "python-jose[cryptography]"`
* **용도:** 로그인 세션 유지 및 사용자 권한(인가) 확인.

```python
from datetime import datetime, timedelta
from jose import jwt, JWTError
from app.core.config import settings

def create_access_token(data: dict):
    """토큰 생성 (HMAC-SHA256 서명 포함)"""
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    
    # settings.JWT_SECRET_KEY를 사용하여 서명 생성
    encoded_jwt = jwt.encode(to_encode, settings.JWT_SECRET_KEY, algorithm=settings.ALGORITHM)
    return encoded_jwt

def verify_access_token(token: str):
    """토큰 검증 (조작 여부 확인)"""
    try:
        # 서명이 일치하지 않거나 만료되면 JWTError 발생
        payload = jwt.decode(token, settings.JWT_SECRET_KEY, algorithms=[settings.ALGORITHM])
        return payload
    except JWTError:
        return None
```

## 요약 및 확인
* **1번(AES-256):** 복호화가 필요한 개인정보 보호용.
* **2번(BCrypt):** 복호화가 필요 없는 비밀번호 저장용.
* **3번(RSA):** 비대칭키 기반의 고도화된 데이터 교환용.
* **4번(HMAC-SHA256):** 로그인 세션의 무결성 및 권한 위조 방지용.
