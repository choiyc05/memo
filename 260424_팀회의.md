```
1. Activity Data 기반 탄소 계산
: 기업의 실제 사용량 데이터를 기준으로 탄소배출량을 산정합니다. **단순 총량이 아니라 실제 사용량을 기반으로 계산하고 싶은데 이 부분에 대한 dummy data를 어떻게 같이 만들어주신다는지가 궁금하다**
CO2e = Activity Data * Emission Factor
예시. 전력 사용량(kWh) * 국가 전력 배출계수

2. PCF 산정
: 제품 단위를 기준으로 탄소배출량을 산정합니다.
PCF = scope 1,2,3 합산 + Logistics

3. CBAM 대응 자동화
: 수출 대상 제품의 내재배출량(Embedded Emissions)을 계산하여 보고서 형태로 자동 생성합니다.
예시. 산출물 - 품목별 배출량, 생산지 정보, 계산 근거 로그
```

## 질문 정리
1. CBAM 범위 설정 관련
```
철강과 알루미늄은 CBAM 대상이지만 품목 분류와 계산 요구 수준이 다른 것으로 이해하고 있습니다.
HS코드 기준으로 대표 제품군 단위로 한정해도 시장성이 있을지 궁금합니다.
한정한다면, 제품 Master를 몇 개 수준으로 가져가는 것이 적절할까요?

제품별 차이를 보여주는 것과 통일성을 갖춘 것 중 어떤 것이 더 경쟁력이 있을지 궁금합니다.
```

2. 제품 세분화 범위 관련 
```
어떤 수준까지 제품 세분화를 실무자 입장에서 유용하다고 생각하시는지 궁금합니다.
```

3. PCF 산정 수준 관련
```
제품 탄소발자국 산정 시 실제 현업에서는 완제품 SKU 단위까지 계산하는지, 아니면 제품군 평균값으로 운영하는 경우도 많은지 궁금합니다.
```
